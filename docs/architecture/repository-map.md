# Repository Map

This document describes the purpose of each top-level folder.

- `data/`: source references and future training corpus manifests (de-identified only).
- `docs/`: architecture, governance, and review documents.
- `instructions/`: reusable safety and drafting primitives.
- `prompts/`: orchestration blueprints for extraction, validation, and composition.
- `scenarios/`: field condition modules used to adapt letter drafts.
- `schemas/`: parameter definitions and validation rules.
- `templates/`: neutral, section-based letter structures.
